# Incorporación en 60 Minutos

Un recorrido estructurado para ser productivo con Synkra AIOS en una hora. Al final, habrás instalado el framework, activado agentes, creado tu primera historia, ejecutado un flujo de trabajo y pasado un gate de calidad.

---

## Minuto 0-10: Instalación y Configuración

### Prerequisitos

Antes de comenzar, asegúrate de tener:

- **Node.js** versión 18.0.0 o superior
- **npm** versión 6.0.0 o superior
- **Git** instalado y configurado
- **GitHub CLI** (`gh`) instalado (para flujos de trabajo de push/PR)
- Un IDE de IA como **Claude Code** (recomendado), Cursor o VS Code con extensiones de IA

### Instalar AIOS

```bash
# Crear un nuevo proyecto
npx @synkra/aios-core init my-first-project

# Navegar a tu proyecto
cd my-first-project
```

### Verificar la Instalación

Verifica que el directorio `.aios-core/` se haya creado con la estructura esperada:

```
my-first-project/
  .aios-core/             # Configuración del framework y agentes
  agents/                 # Directorio de agentes IA
  workflows/              # Flujos de trabajo automatizados
  tasks/                  # Tareas reutilizables
  .env                    # Variables de entorno (configura tu API key)
```

### Configurar Tu Entorno

Edita el archivo `.env` con la clave de tu proveedor de IA:

```env
OPENAI_API_KEY=your-openai-api-key
# o
ANTHROPIC_API_KEY=your-anthropic-api-key
```

## Minuto 10-20: Entendiendo el Meta-Agente

### Iniciar el Meta-Agente

```bash
npx @synkra/aios-core
```

### Conceptos Fundamentales

El **meta-agente** es el corazón de AIOS. Es una IA que comprende su propia base de código y orquesta agentes especializados para manejar diferentes aspectos del desarrollo.

**Capacidades clave:**
- **Auto-Análisis** -- Examina la estructura del código del proyecto
- **Generación de Código** -- Crea componentes siguiendo patrones aprendidos
- **Aprendizaje de Patrones** -- Se adapta al estilo de desarrollo de tu equipo
- **Orquestación de Flujos de Trabajo** -- Coordina agentes a través de flujos de trabajo estructurados

### Comandos Esenciales

Prueba estos comandos para orientarte:

```bash
*help                    # Mostrar todos los comandos disponibles
*status                  # Mostrar estado del sistema
*list-agents             # Mostrar todos los agentes disponibles
*config                  # Ver configuración actual
```

### Los 13 Agentes

AIOS viene con 13 agentes especializados. Los más importantes para conocer de inmediato:

| Agente | Rol | Comando Clave |
|--------|-----|---------------|
| @dev (Dex) | Desarrollo de software | `@dev *develop {story-id}` |
| @sm (River) | Creación de historias | `@sm *draft` |
| @po (Pax) | Validación de historias | `@po *validate-story-draft` |
| @qa | Aseguramiento de calidad | `@qa *qa-gate` |
| @devops (Gage) | Push y despliegue | `@devops *push` |
| @architect (Aria) | Diseño de sistemas | `@architect *create-full-stack-architecture` |
| @pm (Morgan) | Gestión de épicas y proyectos | `@pm *create-epic` |

## Minuto 20-35: Activación de Agentes y Tu Primera Historia

### Activar un Agente

Los agentes se activan dirigiéndote a ellos directamente:

```bash
@sm *help      # Activar el Scrum Master y ver comandos disponibles
@dev *help     # Activar el agente Developer
```

### Entender los Límites de los Agentes

Un concepto importante en AIOS: los agentes tienen **límites de autoridad**.

- @dev puede hacer `git commit` pero **no puede** hacer `git push` (delegar a @devops)
- @po puede validar historias pero **no puede** editar criterios de aceptación después de la validación
- Solo @devops puede crear PRs y enviar al remoto

### Crear Tu Primera Historia

```bash
# Paso 1: Crear un borrador de historia
@sm *draft
```

El Scrum Master te guiará a través de la creación de un archivo de historia con título, descripción, criterios de aceptación y alcance.

```bash
# Paso 2: Validar la historia
@po *validate-story-draft
```

El Product Owner ejecuta el checklist de 10 puntos. Si la puntuación es ≥ 7/10, la historia pasa de `Draft` a `Ready`.

```bash
# Paso 3: Revisar el archivo de historia generado
# Las historias se guardan en docs/stories/
```

## Minuto 35-50: Ejecutando un Flujo de Trabajo (SDC)

### El Story Development Cycle

El SDC es el flujo de trabajo principal en AIOS, con 4 fases:

| Fase | Agente | Qué Sucede |
|------|--------|------------|
| 1. Crear | @sm | Historia borrador desde épica/PRD |
| 2. Validar | @po | Checklist de 10 puntos, decisión GO/NO-GO |
| 3. Implementar | @dev | Código escrito, auto-corrección con CodeRabbit |
| 4. QA Gate | @qa | Verificación de calidad de 7 puntos, veredicto |

### Implementar Tu Historia

```bash
# Iniciar implementación (modo interactivo por defecto)
@dev *develop {story-id}
```

El agente developer trabaja en uno de tres modos:

- **Interactive** (por defecto) -- 5-10 prompts con checkpoints y confirmaciones
- **YOLO** -- Ejecución autónoma con 0-1 prompts; decisiones registradas automáticamente
- **Pre-Flight** -- Todas las preguntas al inicio, luego ejecución sin ambigüedades

Durante la implementación, CodeRabbit escanea automáticamente en busca de problemas:
- Problemas de severidad CRITICAL y HIGH son auto-corregidos (hasta 2 iteraciones)
- Problemas MEDIUM son documentados como deuda técnica
- Problemas LOW son ignorados

## Minuto 50-60: Gates de Calidad y Próximos Pasos

### Ejecutar el QA Gate

```bash
@qa *qa-gate
```

El agente QA verifica 7 dimensiones de calidad:

1. Revisión de código (patrones, legibilidad)
2. Tests unitarios (cobertura, todos pasando)
3. Criterios de aceptación (todos cumplidos)
4. Sin regresiones
5. Rendimiento
6. Seguridad (básicos OWASP)
7. Documentación

**Veredictos posibles:** PASS, CONCERNS, FAIL, WAIVED

### Enviar Tus Cambios

Si el QA gate aprueba:

```bash
# Revisar tus cambios
git diff

# Enviar al remoto (solo @devops puede hacer esto)
@devops *push
```
