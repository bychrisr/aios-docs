# Incorporación en 60 Minutos

Un recorrido estructurado para ser productivo con Synkra AIOS en una hora. Al final, habrás instalado el framework, activado agentes, creado tu primera historia, ejecutado el flujo de trabajo SDC completo y pasado un gate de calidad.

---

## Minuto 0-10: Instalación y Configuración

### Requisitos Previos

Antes de comenzar, asegúrate de tener:

- **Node.js** 20.0.0 o superior
- **npm** 8.0.0 o superior
- **Git** instalado y configurado
- **GitHub CLI** (`gh`) instalado (para flujos de trabajo de push/PR)
- Un IDE de IA — **Claude Code** (recomendado), Cursor o VS Code con extensiones de IA

### Instalar AIOS

```bash
# Crear un nuevo proyecto
npx @synkra/aios-core init my-first-project

# Navegar a tu proyecto
cd my-first-project
```

### Verificar la Instalación

Comprueba que el directorio `.aios-core/` se haya creado:

```
my-first-project/
  .aios-core/          # Configuración del framework, agentes, tareas, flujos de trabajo
  docs/
    stories/           # Los archivos de historia se guardarán aquí
```

### Configurar Tu Entorno

Edita el archivo `.env` con la clave de tu proveedor de IA:

```env
ANTHROPIC_API_KEY=your-anthropic-api-key
# o
OPENAI_API_KEY=your-openai-api-key
```

## Minuto 10-20: Entendiendo el Framework

### Conceptos Fundamentales

AIOS orquesta 12 agentes especializados con nombre a través de flujos de trabajo estructurados. Cada agente tiene:

- Un **handle** (ej. `@dev`) que usas para dirigirte a él
- Un **nombre** (ej. Dex) para su persona
- **Límites de autoridad** definidos — qué puede y no puede hacer
- Un conjunto de **comandos** a los que responde

### Comandos Esenciales (Cualquier Agente)

```bash
*help      # Mostrar todos los comandos disponibles
*status    # Mostrar el contexto actual, el estado de la historia y el agente
*guide     # Mostrar la guía de uso detallada para el agente activo
```

### Los 12 Agentes

| Handle | Nombre | Rol |
|--------|--------|-----|
| `@dev` | Dex | Implementación de software |
| `@qa` | Quinn | Gates de calidad |
| `@po` | Pax | Validación de historias |
| `@pm` | Morgan | Épicas y spec pipeline |
| `@sm` | River | Creación de historias |
| `@architect` | Aria | Arquitectura de sistemas |
| `@analyst` | Atlas | Investigación |
| `@ux-design-expert` | Uma | UX/Diseño |
| `@data-engineer` | Dara | Base de datos |
| `@devops` | Gage | CI/CD y Git push |
| `@docs` | Ajax | Documentación |
| `@aios-master` | Orion | Orquestación del framework |

Para detalles completos sobre cada agente, consulta [Agentes](/playbook/agents).

## Minuto 20-35: Activación de Agentes y Tu Primera Historia

### Activar un Agente

Dirígete a los agentes directamente para activarlos:

```bash
@sm *help      # Activar a River y ver los comandos disponibles
@dev *help     # Activar a Dex
```

### Límites de Autoridad de los Agentes

Un concepto crítico de AIOS: los agentes tienen límites de autoridad estrictos.

- @dev puede ejecutar `git add`, `git commit` — pero **no puede** hacer `git push` (delegar a @devops)
- @po puede validar historias — pero **no puede** editar los criterios de aceptación tras la validación
- Solo @devops (Gage) puede ejecutar `git push`, `gh pr create` y `gh pr merge`

### Fase 1: Crear Tu Primera Historia (@sm — River)

```bash
@sm *draft
```

River te guía para crear una historia con título, descripción, criterios de aceptación y alcance. El resultado es un archivo `{epicNum}.{storyNum}.story.md` con estado `Draft`.

### Fase 2: Validar la Historia (@po — Pax)

```bash
@po *validate-story-draft
```

Pax ejecuta el **checklist de validación de 10 puntos**:

| # | Verificación |
|---|-------------|
| 1 | Título claro y objetivo |
| 2 | Descripción completa |
| 3 | Criterios de aceptación verificables |
| 4 | Alcance bien definido (IN/OUT) |
| 5 | Dependencias mapeadas |
| 6 | Estimación de complejidad |
| 7 | Valor de negocio declarado |
| 8 | Riesgos documentados |
| 9 | Definition of Done |
| 10 | Alineación con PRD/Épica |

**Decisión:**
- **GO** (≥ 7/10) — El estado de la historia cambia de `Draft` a `Ready`
- **NO-GO** (menos de 7) — Correcciones requeridas listadas; la historia permanece en `Draft`

## Minuto 35-50: Ejecutando el Story Development Cycle (SDC)

### Las 4 Fases del SDC

| Fase | Agente | Qué Sucede |
|------|--------|------------|
| 1. Crear | @sm (River) | Historia creada desde épica/PRD |
| 2. Validar | @po (Pax) | Checklist de 10 puntos, GO/NO-GO |
| 3. Implementar | @dev (Dex) | Código escrito, auto-corrección con CodeRabbit |
| 4. QA Gate | @qa (Quinn) | Verificación de calidad de 7 puntos, veredicto |

### Fase 3: Implementar (@dev — Dex)

```bash
@dev *develop story-1.1
```

Dex trabaja en uno de tres modos de ejecución:

| Modo | Prompts | Mejor Para |
|------|---------|-----------|
| **Interactive** (por defecto) | 5-10, con checkpoints | Aprendizaje, decisiones complejas |
| **YOLO** | 0-1, totalmente autónomo | Tareas simples |
| **Pre-Flight** | 10-15 al inicio, luego cero | Requisitos ambiguos |

Durante la implementación, **CodeRabbit** escanea automáticamente:
- Los problemas CRITICAL y HIGH son auto-corregidos (hasta 2 iteraciones en la fase dev)
- Los problemas MEDIUM se documentan como deuda técnica
- Los problemas LOW se ignoran

## Minuto 50-60: Gates de Calidad y Publicación

### Fase 4: QA Gate (@qa — Quinn)

```bash
@qa *qa-gate
```

Quinn verifica 7 dimensiones de calidad:

| # | Verificación |
|---|-------------|
| 1 | Revisión de código (patrones, legibilidad) |
| 2 | Tests unitarios (cobertura, todos pasando) |
| 3 | Criterios de aceptación (todos cumplidos) |
| 4 | Sin regresiones |
| 5 | Rendimiento dentro de límites |
| 6 | Seguridad (básicos OWASP) |
| 7 | Documentación (actualizada si es necesario) |

**Veredictos posibles:** PASS, CONCERNS, FAIL, WAIVED

### Si FAIL — Iniciar el QA Loop

```bash
*qa-loop story-1.1       # Iniciar ciclo automatizado de revisión-corrección
# Máx. 5 iteraciones: @qa revisa → @dev corrige → @qa vuelve a revisar
```

### Publicar Tus Cambios (@devops — Gage)

```bash
@devops *push
```

Solo Gage puede hacer push. Esto es por diseño — todos los agentes coordinan a través de esta única puerta para las operaciones remotas.

### Lo Que Lograste

En 60 minutos has:

- Instalado y configurado Synkra AIOS
- Aprendido los 12 agentes, sus nombres y límites de autoridad
- Creado y validado tu primera historia (Fases 1-2 del SDC)
- Implementado con auto-corrección de CodeRabbit (Fase 3)
- Pasado un gate de calidad (Fase 4)
- Publicado tus cambios a través de @devops
