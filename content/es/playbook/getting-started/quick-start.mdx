# Guía de Inicio Rápido

Empieza a usar AIOS en 5 pasos.

## Paso 1: Inicializar AIOS

En la raíz de tu proyecto:

```bash
npx @synkra/aios-core init
```

Esto crea el directorio `.aios-core/` con todos los archivos del framework, agentes, tareas y flujos de trabajo.

## Paso 2: Activar Tu Primer Agente

En Claude Code (o tu IDE de IA):

```
@dev
```

Esto activa a Dex, el agente Developer, con todas las capacidades de desarrollo. Puedes dirigirte a cualquiera de los 12 agentes por su handle.

## Paso 3: Ejecutar Tu Primer Comando

```
*help
```

Muestra todos los comandos disponibles para el agente activo. Todos los agentes admiten `*help`, `*status` y `*guide`.

## Paso 4: Crear Tu Primera Historia

Activa a River, el Scrum Master, y crea una historia:

```
@sm *draft
```

Sigue los prompts interactivos para crear tu primera historia de desarrollo. El resultado es un archivo `{epicNum}.{storyNum}.story.md` con estado `Draft`.

Luego valídala con Pax, el Product Owner:

```
@po *validate-story-draft
```

Una puntuación de ≥ 7/10 mueve la historia de `Draft` a `Ready`.

## Paso 5: Empezar a Desarrollar

Con una historia en estado `Ready`:

```
@dev *develop story-1.1
```

Dex te guía a través de la implementación en modo Interactive por defecto. CodeRabbit escanea automáticamente y auto-corrige los problemas CRITICAL y HIGH.

Cuando la implementación esté completa:

```
@qa *qa-gate
```

Quinn ejecuta el gate de calidad de 7 puntos. Al obtener PASS, publica con:

```
@devops *push
```

Solo Gage (@devops) puede hacer push — todos los demás agentes le delegan las operaciones de push.

---

## Comandos Esenciales de un Vistazo

| Comando | Agente | Descripción |
|---------|--------|-------------|
| `*help` | Cualquiera | Mostrar comandos disponibles |
| `@sm *draft` | River | Crear una nueva historia |
| `@po *validate-story-draft` | Pax | Validar una historia (checklist de 10 puntos) |
| `@dev *develop {id}` | Dex | Iniciar la implementación de una historia |
| `@qa *qa-gate` | Quinn | Ejecutar verificaciones de calidad (gate de 7 puntos) |
| `@devops *push` | Gage | Enviar cambios al remoto |
