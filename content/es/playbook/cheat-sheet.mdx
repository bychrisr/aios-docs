# Cheat Sheet

Secuencias de comandos listas para copiar para el uso diario de AIOS. Para explicaciones completas, consulta [Comandos](/playbook/commands) o [Agentes](/playbook/agents).

---

## Comandos Esenciales del Día a Día

```
*help          # Muestra los comandos disponibles para el agente activo
*status        # Muestra el contexto actual, la historia y el estado del agente
*guide         # Muestra la guía de uso detallada para el agente activo
```

---

## Story Development Cycle Completo (SDC)

El flujo de trabajo completo de 4 fases desde la creación de la historia hasta el envío:

```
# Fase 1 — Crear (Scrum Master)
@sm *draft

# Fase 2 — Validar (Product Owner)
@po *validate-story-draft
# GO = ≥ 7/10 → el estado pasa de Draft → Ready
# NO-GO = < 7/10 → corregir y reintentar

# Fase 3 — Implementar (Developer)
@dev *develop {story-id}

# Fase 4 — QA Gate (Agente QA)
@qa *qa-gate
# Veredictos: PASS | CONCERNS | FAIL | WAIVED

# Publicar (DevOps — solo @devops puede hacer push)
@devops *push
```

---

## QA y Calidad

```
@qa *qa-gate                   # Ejecutar el gate de calidad de 7 puntos en la historia actual
*qa-loop {storyId}             # Iniciar ciclo automatizado de revisión-corrección (máx. 5 iteraciones)
*qa-loop-review                # Reanudar el ciclo desde el paso de revisión
*qa-loop-fix                   # Reanudar el ciclo desde el paso de corrección
*stop-qa-loop                  # Pausar el ciclo y guardar el estado
*resume-qa-loop                # Reanudar el ciclo desde el estado guardado
*escalate-qa-loop              # Forzar escalamiento manual
```

## Planificación y Épicas

```
@pm *create-epic               # Crear una nueva épica a partir de requisitos
@pm *execute-epic              # Ejecutar una épica (activa el flujo de creación de historias)
@pm *create-doc prd            # Generar un documento PRD desde plantilla
@sm *draft                     # Crear una nueva historia desde épica/PRD
@po *validate-story-draft      # Validar la historia (checklist de 10 puntos)
```

## IDS (Sistema de Desarrollo Incremental)

```
*ids check {intent}            # ¿Esto debe ser REUSE, ADAPT o CREATE?
*ids impact {entity-id}        # Analizar el impacto de modificar una entidad
*ids register {file-path}      # Registrar una nueva entidad en el registro IDS
*ids health                    # Verificación de salud del registro IDS
```

## Modos de Ejecución

Al llamar a `@dev *develop`, puedes especificar el modo de ejecución:

| Modo | Prompts | Mejor Para |
|------|---------|-----------|
| **Interactive** (por defecto) | 5-10, con checkpoints | Aprendizaje, decisiones complejas |
| **YOLO** | 0-1, totalmente autónomo | Tareas simples, contexto confiable |
| **Pre-Flight** | 10-15 al inicio, luego cero | Requisitos ambiguos, trabajo crítico |

```
@dev *develop {story-id}              # Interactive (por defecto)
@dev *develop {story-id} --yolo       # Modo YOLO
@dev *develop {story-id} --pre-flight # Modo Pre-Flight
```

## Progresión de Estado de la Historia

```
Draft → Ready → InProgress → InReview → Done
```

| Transición | Quién | Cómo |
|------------|-------|------|
| Draft → Ready | @po | `*validate-story-draft` (veredicto GO) |
| Ready → InProgress | @dev | `*develop {story-id}` |
| InProgress → InReview | @qa | `*qa-gate` |
| InReview → Done | @devops | `*push` |

## Auto-corrección de CodeRabbit (Fase Dev)

AIOS ejecuta CodeRabbit automáticamente durante `@dev *develop`. No necesitas invocarlo manualmente.

| Severidad | Fase Dev | Fase QA |
|-----------|----------|---------|
| CRITICAL | Auto-fix (máx. 2 iteraciones) | Auto-fix (máx. 3 iteraciones) |
| HIGH | Auto-fix o documentar como deuda | Auto-fix o documentar como deuda |
| MEDIUM | Documentar como deuda técnica | Documentar como deuda técnica |
| LOW | Ignorado | Ignorado |
