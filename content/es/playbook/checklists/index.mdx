# Listas de Verificación

Listas de verificación de calidad para cada fase del desarrollo AIOS.

## Validación de Historia (10 Puntos)

1. Título claro y objetivo
2. Descripción completa (problema/necesidad explicada)
3. Criterios de aceptación comprobables (Given/When/Then preferido)
4. Alcance bien definido (IN y OUT claramente listados)
5. Dependencias mapeadas (historias/recursos prerequisito)
6. Estimación de complejidad (puntos o sizing T-shirt)
7. Valor del negocio (beneficio para usuario/negocio claro)
8. Riesgos documentados (problemas potenciales identificados)
9. Criterios de Done (definición clara de completo)
10. Alineación con PRD/Épica (consistencia con documentos fuente)

**Puntuación de Aprobación:** ≥ 7/10

## QA Gate (7 Verificaciones)

1. Revisión de código — patrones, legibilidad, mantenibilidad
2. Tests unitarios — cobertura adecuada, todos pasando
3. Criterios de aceptación — todos cumplidos según AC de historia
4. Sin regresiones — funcionalidad existente preservada
5. Rendimiento — dentro de límites aceptables
6. Seguridad — conceptos básicos OWASP verificados
7. Documentación — actualizada si es necesario
