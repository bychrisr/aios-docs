# Comandos de Agentes Especializados

Cada agente de AIOS tiene su propio conjunto de comandos para su dominio. Usa el handle del agente para dirigirte a ellos directamente.

Para una referencia rápida en una sola página, consulta el [Cheat Sheet](/playbook/cheat-sheet). Para personas y roles de agentes, consulta [Agentes](/playbook/agents).

---

## @sm — River (Scrum Master)

| Comando | Descripción |
|---------|-------------|
| `@sm *draft` | Crear una nueva historia (creación interactiva guiada) |
| `@sm *create-story` | Comando alternativo de creación de historias |
| `@sm *help` | Mostrar todos los comandos de @sm |

---

## @po — Pax (Product Owner)

| Comando | Descripción |
|---------|-------------|
| `@po *validate-story-draft` | Ejecutar el checklist de validación de 10 puntos; decisión GO/NO-GO |
| `@po *help` | Mostrar todos los comandos de @po |

**Decisión:** GO (≥ 7/10) mueve la historia de `Draft` a `Ready`.

---

## @dev — Dex (Developer)

| Comando | Descripción |
|---------|-------------|
| `@dev *develop {story-id}` | Iniciar la implementación de una historia (modo Interactive) |
| `@dev *develop {story-id} --yolo` | Modo YOLO — autónomo, 0-1 prompts |
| `@dev *develop {story-id} --pre-flight` | Pre-Flight — todas las preguntas al inicio |
| `@dev *help` | Mostrar todos los comandos de @dev |

**Autoridad:** Puede ejecutar `git add`, `git commit`, `git branch`, `git checkout`. No puede hacer `git push` — delegar a @devops.

## @qa — Quinn (QA Engineer)

| Comando | Descripción |
|---------|-------------|
| `@qa *qa-gate` | Ejecutar el gate de calidad de 7 puntos |
| `*qa-loop {storyId}` | Iniciar ciclo automatizado de revisión-corrección (máx. 5 iteraciones) |
| `*qa-loop-review` | Reanudar el ciclo desde el paso de revisión |
| `*qa-loop-fix` | Reanudar el ciclo desde el paso de corrección |
| `*stop-qa-loop` | Pausar el ciclo y guardar el estado en `qa/loop-status.json` |
| `*resume-qa-loop` | Reanudar desde el estado guardado |
| `*escalate-qa-loop` | Forzar escalamiento manual |
| `@qa *help` | Mostrar todos los comandos de @qa |

**Veredictos del QA gate:** PASS, CONCERNS, FAIL, WAIVED

## Reglas de Delegación

AIOS impone patrones de delegación estrictos. Reglas clave:

- Cualquier agente que necesite publicar código DEBE delegar a `@devops *push`
- @architect decide la tecnología; @data-engineer implementa el DDL
- @sm crea historias; @po las valida — ninguno puede hacer el trabajo del otro
- @aios-master (Orion) puede anular los límites cuando sea necesario para la salud del framework
