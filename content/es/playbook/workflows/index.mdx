# Guías de Workflow

Guías paso a paso para flujos de trabajo comunes en AIOS. Cada guía explica cuándo usar el flujo de trabajo, qué agentes están involucrados y los comandos a ejecutar en cada etapa.

## Flujos de Trabajo de Proyecto

Estos flujos de trabajo cubren ciclos de vida de proyecto de extremo a extremo:

- **[Proyecto Greenfield](/playbook/workflows/greenfield-project)** -- Comenzar un nuevo proyecto desde cero. Cubre las tres variantes: full-stack, servicio/API y solo UI, incluyendo bootstrap del entorno, descubrimiento, planificación y el ciclo de desarrollo.

- **[Proyecto Brownfield](/playbook/workflows/brownfield-project)** -- Unirse o mejorar una base de código existente. Cubre la evaluación Brownfield Discovery de 10 fases y las tres variantes de implementación para proyectos existentes.

## Flujos de Trabajo de Desarrollo

Estos flujos de trabajo cubren el proceso de desarrollo diario:

- **[Planificación de Sprint](/playbook/workflows/sprint-planning)** -- Planificar y ejecutar sprints semanales. Cubre creación de épicas, validación de historias, estimación, seguimiento de velocidad y retrospectivas.

- **[Flujo de PR](/playbook/workflows/pr-workflow)** -- El proceso estándar de pull request. Cubre push/merge de @devops, integración con CodeRabbit, bucles de auto-corrección y decisiones del QA gate.

## Flujos de Trabajo Operacionales

- **[Gestión de Crisis](/playbook/workflows/crisis-management)** -- Procedimientos de sala de guerra para problemas críticos de producción. Cubre clasificación de severidad, disparadores de escalamiento, flujos de trabajo de hotfix, procedimientos de rollback y documentación post-mortem.

## Elegir el Flujo de Trabajo Correcto

| Situación | Flujo de Trabajo Recomendado |
|-----------|------------------------------|
| Nuevo proyecto, sin código existente | [Proyecto Greenfield](/playbook/workflows/greenfield-project) |
| Proyecto existente, necesita evaluación | [Proyecto Brownfield](/playbook/workflows/brownfield-project) (Discovery) |
| Agregar funcionalidades a proyecto existente | [Proyecto Brownfield](/playbook/workflows/brownfield-project) (Implementación) |
| Cadencia de desarrollo semanal | [Planificación de Sprint](/playbook/workflows/sprint-planning) |
| Código listo para enviar | [Flujo de PR](/playbook/workflows/pr-workflow) |
| Incidente de producción | [Gestión de Crisis](/playbook/workflows/crisis-management) |
| Corrección de bug simple | [Planificación de Sprint](/playbook/workflows/sprint-planning) (SDC en modo YOLO) |
