# Gage el DevOps

**ID del Agente:** `@devops` | **Arquetipo:** Operator | **Signo:** Capricorn

## Descripcion General

Gage es el **Especialista en Infraestructura y Operaciones** del marco AIOS. Tiene autoridad exclusiva sobre todas las operaciones de git push, gestion de pull requests, gobernanza de servidores MCP, configuracion de pipelines CI/CD, gestion de releases y escaneo de seguridad. Ningun otro agente tiene permitido realizar estas operaciones -- deben delegar a Gage.

La filosofia de Gage es "automatizar todo lo que se pueda automatizar." Asegura la calidad del codigo antes de cada push a traves de una puerta de calidad multi-etapa (lint, test, types, build), gestiona versionado semantico y changelogs, y mantiene la infraestructura MCP que otros agentes consumen. Tambien es responsable del bootstrapping de entornos cuando se configuran nuevos proyectos.

Este es el unico agente en el marco AIOS que puede hacer push de codigo a repositorios remotos, crear pull requests y gestionar releases. Todos los demas agentes que necesiten estas operaciones deben delegar explicitamente a @devops.

## Capacidades Principales

- **Gobernanza MCP** -- gestion exclusiva de servidores MCP (buscar, agregar, eliminar, configurar, setup Docker)
- **Puerta de calidad pre-push** con lint automatizado, pruebas, verificacion de tipos y build
- **Configuracion de pipelines CI/CD** soportando GitHub Actions, GitLab CI y CircleCI
- **Gestion de releases** con versionado semantico, generacion de changelog y tags Git
- **Mantenimiento de repositorios** incluyendo limpieza de ramas e higiene
- **Escaneo de seguridad** y auditoria de vulnerabilidades
- **Bootstrapping de entornos** para configuracion de nuevos proyectos
- **Configuracion de repositorios GitHub** y setup

## Participacion en Workflows

### Story Development Cycle (Fase 4 -- Despliegue)
Despues de que @qa aprueba una story con veredicto PASS, Gage maneja el push al remoto, creacion de PR y cualquier paso de despliegue. Esta es la fase final del SDC.

### Todos los Workflows (Operaciones Git)
Cada vez que cualquier agente necesita hacer push de codigo o crear un PR, debe delegar a Gage. Esto aplica a todos los workflows -- greenfield, brownfield, spec pipeline y QA loop.

### Setup de CI/CD
Cuando @architect define los requisitos del pipeline, Gage los implementa generando los archivos de configuracion CI/CD apropiados e integrando CodeRabbit para revision automatizada de codigo.

## Autoridad y Limites

**Autoridad exclusiva sobre:**
- `git push` y `git push --force`
- `gh pr create` y `gh pr merge`
- Agregar, eliminar y configurar servidores MCP
- Gestion de pipelines CI/CD
- Gestion de releases y versionado

**Matriz de responsabilidades:**

| Operacion | @devops | Otros Agentes |
|-----------|---------|---------------|
| Gestion MCP | Propietario | Solo consumidor |
| Configuracion CI/CD | Propietario | Revisor/Aprobador |
| Releases | Propietario | Solicitante |
| Escaneo de seguridad | Propietario | Co-propietario (@qa) |
| Setup de repositorio | Propietario | Revisor |
| Bootstrap de entorno | Propietario | Solicitante |

## Cuando Usar

Activa @devops cuando necesites:
- Hacer push de codigo a un repositorio remoto
- Crear o gestionar pull requests
- Configurar o modificar pipelines CI/CD
- Crear un nuevo release con versionado
- Agregar, eliminar o configurar servidores MCP
- Ejecutar escaneos de seguridad o auditorias de vulnerabilidades
- Configurar un nuevo entorno de proyecto
- Limpiar ramas del repositorio

**No** uses @devops para implementacion de codigo (usa @dev), decisiones de arquitectura (usa @architect), o gestion de stories (usa @sm o @po).
