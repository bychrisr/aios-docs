# Orion -- AIOS Master

**Arquetipo:** Orchestrator | **ID del Agente:** `@aios-master`

Orion es el meta-agente y orquestador universal del marco AIOS. Como AIOS Master, Orion puede ejecutar cualquier tarea de cualquier agente directamente, crear y modificar componentes del marco (agentes, tareas, workflows, plantillas) y coordinar workflows complejos multi-agente. Orion tambien gestiona la Base de Conocimiento AIOS y sirve como autoridad de gobernanza del marco.

## Capacidades Principales

- **Orquestacion Universal** -- Ejecutar cualquier tarea de cualquier agente sin transformacion de persona
- **Desarrollo del Marco** -- Crear, modificar y deprecar agentes, tareas, workflows y plantillas
- **Gestion de Componentes** -- Validar, analizar y gestionar el ciclo de vida de todos los componentes del marco
- **Coordinacion Multi-Agente** -- Orquestar workflows complejos que involucran multiples agentes
- **Base de Conocimiento** -- Acceder a la base de conocimiento completa del Metodo AIOS via `*kb`
- **Correccion de Rumbo** -- Analizar y corregir desviaciones de proceso o calidad con `*correct-course`

## Comandos Clave

### Comandos de Creacion

| Comando | Descripcion |
|---------|-------------|
| `*create agent {name}` | Crear una nueva definicion de agente usando el sistema de plantillas |
| `*create task {name}` | Crear un nuevo archivo de tarea estandarizado |
| `*create workflow {name}` | Crear una nueva definicion de workflow multi-agente |
| `*create-doc {template}` | Crear un documento a partir de una plantilla YAML |
| `*create-next-story` | Crear la siguiente user story |

### Comandos de Modificacion

| Comando | Descripcion |
|---------|-------------|
| `*modify agent {name}` | Modificar un agente existente con respaldo y rollback |
| `*modify task {name}` | Modificar una tarea existente preservando la compatibilidad |
| `*modify workflow {name}` | Modificar un workflow existente manteniendo la integridad |
| `*update-manifest` | Actualizar el manifiesto del equipo con nuevos agentes |
| `*propose-modification` | Crear una propuesta de modificacion para revision colaborativa |

### Comandos de Analisis

| Comando | Descripcion |
|---------|-------------|
| `*analyze-framework` | Analizar la estructura del marco, redundancias y rendimiento |
| `*deprecate-component` | Deprecar un componente con cronograma y plan de migracion |
| `*correct-course` | Analizar y corregir desviaciones de proceso/calidad |
| `*execute-checklist {name}` | Ejecutar una lista de verificacion de validacion |

### Comandos de Ejecucion

| Comando | Descripcion |
|---------|-------------|
| `*task {name}` | Ejecutar una tarea especifica directamente |
| `*workflow {name}` | Iniciar un workflow multi-agente |
| `*kb` | Activar/desactivar el modo interactivo de la Base de Conocimiento |
| `*help` | Mostrar todos los comandos disponibles |
| `*status` | Mostrar el contexto y progreso actual |

## Participacion en Workflows

Orion participa y orquesta todos los workflows principales de AIOS:

- **Story Development Cycle (SDC)** -- Puede ejecutar cualquier fase directamente
- **Spec Pipeline** -- Coordina las fases de investigacion, evaluacion y planificacion
- **Brownfield Discovery** -- Orquesta el workflow de evaluacion de 10 fases
- **QA Loop** -- Puede intervenir cuando se activa la escalacion
- **Workflows Greenfield y Brownfield** -- Variantes full-stack, servicio y UI

## Cuando Usar Este Agente

**Usa @aios-master cuando necesites:**
- Crear o modificar componentes del marco (agentes, tareas, workflows)
- Orquestar workflows complejos multi-agente
- Ejecutar cualquier tarea directamente sin cambiar de agente
- Realizar operaciones del meta-marco y coordinacion entre agentes
- Acceder a la Base de Conocimiento AIOS (`*kb`)
- Analizar y mejorar el propio marco

**No uses @aios-master cuando:**
- Un agente especializado seria mas apropiado (ej., @dev para codificacion, @qa para revisiones)
- Solo necesitas una tarea de un solo agente -- usa el especialista relevante en su lugar
