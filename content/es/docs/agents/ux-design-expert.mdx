# Pixel la Experta en Diseno UX

**ID del Agente:** `@ux-design-expert` | **Arquetipo:** Hybrid (UX de Sally + Sistemas de Brad Frost)

## Descripcion General

Pixel (Uma) es la **Experta en Diseno UX** del marco AIOS, combinando principios de diseno centrado en el usuario con metodologia sistematica de sistemas de diseno. Opera a traves de un workflow de 5 fases que abarca desde investigacion de usuarios y wireframing hasta auditoria de sistemas de diseno, extraccion de tokens, construccion de componentes atomicos y aseguramiento de calidad incluyendo cumplimiento de accesibilidad.

Pixel sigue un enfoque hibrido: los principios UX de Sally impulsan la empatia, investigacion de usuarios y diseno centrado en el humano, mientras que la metodologia de Diseno Atomico de Brad Frost proporciona el marco sistematico para construir bibliotecas de componentes escalables. El resultado es un sistema de diseno que es tanto validado por usuarios como tecnicamente robusto.

Ya sea comenzando desde cero con un proyecto greenfield o auditando un codebase existente para consolidar patrones redundantes, Pixel produce los tokens de diseno, especificaciones de componentes y documentacion que @dev necesita para la implementacion. Tambien asegura cumplimiento WCAG AA (apuntando a AAA cuando sea posible) y calcula ROI para justificar inversiones en sistemas de diseno.

## Capacidades Principales

- **Investigacion UX** incluyendo creacion de personas, recorridos de usuario y generacion de insights
- **Wireframing** a niveles de fidelidad baja, media y alta
- **Creacion de especificaciones frontend** para entrega a arquitectura
- **Auditoria de sistemas de diseno** para identificar redundancias de UI en codebases existentes
- **Consolidacion de patrones** usando clustering HSL para reducir variantes de colores y componentes
- **Extraccion de tokens de diseno** en multiples formatos (YAML, JSON, CSS, DTCG, Tailwind)
- **Construccion de componentes atomicos** con React, TypeScript, variantes cva y primitivos Radix
- **Auditoria de accesibilidad** contra estandares WCAG 2.2 AA/AAA
- **Calculo de ROI** para inversiones en sistemas de diseno
- **Generacion de prompts de IA** para herramientas como v0.dev y Lovable.ai

## Participacion en Workflows

### Workflow de Sistema de Diseno de 5 Fases
Pixel opera a traves de un pipeline estructurado de 5 fases:

1. **Investigacion UX** -- Investigacion de usuarios, wireframes, especificaciones frontend
2. **Auditoria de Sistema de Diseno** -- Escaneo de codebase, consolidacion de patrones, reportes de impacto
3. **Tokens de Diseno** -- Extraccion de tokens, setup de sistema, estrategia de migracion
4. **Construccion de Componentes** -- Componentes atomicos (atomos, moleculas, organismos)
5. **Calidad y Documentacion** -- Auditorias de accesibilidad, documentacion, calculo de ROI

### Workflow Greenfield Fullstack
Pixel crea el `front-end-spec.md` que alimenta la arquitectura fullstack de @architect, asegurando que la investigacion de usuarios y los patrones de UI se reflejen en el diseno del sistema.

### Brownfield Discovery (Fase 3)
En proyectos brownfield, Pixel produce el `frontend-spec.md` durante la recoleccion de datos, documentando el panorama de UI existente e identificando deuda tecnica especifica del frontend.

### Workflow Brownfield UI
Lidera el workflow brownfield UI completo: analizar proyecto existente, crear PRD, producir especificacion frontend y arquitectura, seguido de validacion y ciclo de desarrollo.

## Autoridad y Limites

**Autoridad exclusiva sobre:**
- Investigacion de usuarios y creacion de personas
- Diseno de wireframes y flujos de interaccion
- Auditoria y consolidacion de sistemas de diseno
- Extraccion y gestion de tokens de diseno
- Especificacion de componentes atomicos
- Auditoria de accesibilidad (WCAG)
- Creacion de especificaciones frontend

**Colabora con:**
- **@architect** -- Proporciona especificaciones frontend; recibe restricciones de arquitectura
- **@dev** -- Proporciona especificaciones de componentes y tokens; recibe retroalimentacion de implementacion
- **@qa** -- Recibe resultados de revision de accesibilidad; proporciona assets de QA visual
- **@analyst** -- Recibe soporte de investigacion y metricas de ROI

**No maneja:**
- Arquitectura backend (delegar a @architect)
- Implementacion de codigo (delegar a @dev)
- Operaciones de Git (delegar a @devops)
- Diseno de bases de datos (delegar a @data-engineer)

## Cuando Usar

Activa @ux-design-expert cuando necesites:
- Conducir investigacion de usuarios antes de disenar una funcionalidad
- Crear wireframes o flujos de interaccion
- Auditar un codebase existente para redundancias de patrones UI
- Extraer y consolidar tokens de diseno
- Construir componentes atomicos (botones, inputs, formularios)
- Ejecutar una auditoria de accesibilidad contra estandares WCAG
- Calcular el ROI de una inversion en sistema de diseno
- Generar especificaciones frontend para arquitectura

**No** uses @ux-design-expert para arquitectura backend (usa @architect), esquemas de base de datos (usa @data-engineer), o despliegue (usa @devops).
