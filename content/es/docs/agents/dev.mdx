# Dex -- Desarrollador

**Arquetipo:** Builder | **ID del Agente:** `@dev`

Dex es el Desarrollador Full Stack del marco AIOS. Responsable de implementar user stories, depurar, refactorizar y aplicar buenas practicas de desarrollo. Dex opera con un enfoque pragmatico y orientado a soluciones, implementando stories con precision mientras mantiene una cobertura de pruebas integral.

## Capacidades Principales

- **Implementacion de Stories** -- Desarrollar user stories completas con tres modos de ejecucion (YOLO, Interactive, Pre-flight)
- **Calidad de Codigo** -- Mejorar formateo, linting y patrones de sintaxis moderna
- **Optimizacion de Rendimiento** -- Analizar y optimizar el rendimiento del codigo
- **Refactorizacion** -- Sugerir y aplicar oportunidades de refactorizacion automatizada
- **Aplicacion de Correcciones QA** -- Aplicar correcciones basadas en retroalimentacion de QA
- **Gestion de Deuda Tecnica** -- Registrar y gestionar deuda tecnica en el backlog
- **Auto-Correccion CodeRabbit** -- Revision automatizada de codigo con ciclo de auto-correccion (maximo 2 iteraciones)

## Comandos Clave

### Comandos de Desarrollo

| Comando | Descripcion |
|---------|-------------|
| `*develop {story-id}` | Implementar una story completa (modo por defecto: Interactive) |
| `*develop {story-id} yolo` | Implementar autonomamente con 0-1 prompts |
| `*develop {story-id} preflight` | Planificar todo de antemano, luego ejecutar sin ambiguedad |
| `*run-tests` | Ejecutar linting y pruebas |

### Comandos de Calidad

| Comando | Descripcion |
|---------|-------------|
| `*apply-qa-fixes` | Aplicar correcciones de la retroalimentacion de QA |
| `*improve-code-quality {path}` | Mejorar calidad del codigo (formateo, linting, sintaxis moderna) |
| `*optimize-performance {path}` | Analizar y optimizar rendimiento del codigo |
| `*suggest-refactoring {path}` | Sugerir oportunidades de refactorizacion automatizada |

### Backlog y Documentacion

| Comando | Descripcion |
|---------|-------------|
| `*backlog-debt` | Registrar deuda tecnica en el backlog |
| `*sync-documentation` | Sincronizar documentacion con cambios de codigo |
| `*validate-story-draft` | Validar calidad y completitud del borrador de story |

## Modos de Ejecucion

| Modo | Cuando Usar | Prompts |
|------|-------------|---------|
| **YOLO** | Tareas simples y deterministas | 0-1 |
| **Interactive** (por defecto) | Oportunidades de aprendizaje, decisiones complejas | 5-10 |
| **Pre-flight** | Requisitos ambiguos, trabajo critico | Todos de antemano |

## Participacion en Workflows

- **Story Development Cycle (SDC)** -- Fase 3: Implementacion. Recibe stories validadas de @po, implementa codigo, escribe pruebas y pasa a @qa para revision
- **QA Loop** -- Recibe solicitudes de correccion de @qa, aplica correcciones y envia para re-revision
- **Workflows Greenfield y Brownfield** -- Participa en las 6 variantes de workflow (full-stack, servicio, UI)

## Permisos de Git

**Operaciones permitidas:** `git add`, `git commit`, `git status`, `git diff`, `git log`, `git branch`, `git checkout`, `git merge`

**Operaciones bloqueadas (delegar a @devops):** `git push`, `git push --force`, `gh pr create`, `gh pr merge`

## Cuando Usar Este Agente

**Usa @dev cuando necesites:**
- Implementar user stories aprobadas
- Aplicar solicitudes de correccion de QA
- Refactorizar codigo existente
- Optimizar rendimiento
- Registrar deuda tecnica
- Ejecutar y validar pruebas

**No uses @dev cuando:**
- Crees stories (usa @sm)
- Hagas push al remoto (usa @devops)
- Valides arquitectura (usa @architect)
- Gestiones prioridades del backlog (usa @po)
