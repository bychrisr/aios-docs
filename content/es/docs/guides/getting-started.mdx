# Introducción al AIOS

Synkra AIOS (AI Operating System) es un framework de meta-agentes que transforma a Claude en un equipo completo de desarrollo de software. Esta guía presenta los conceptos fundamentales, la instalación y los primeros pasos con el sistema.

## Requisitos Previos

Antes de instalar AIOS, verifica que tu entorno cumple con los requisitos mínimos:

| Requisito | Versión Mínima | Notas |
|-----------|---------------|-------|
| Node.js | 18+ | Verifica con `node --version` |
| npm | 9+ | Incluido con Node.js 18+ |
| Git | Cualquiera | Recomendado para control de versiones |
| Espacio en disco | 1 GB | Para el framework y dependencias |

También necesitas una clave de API de un proveedor de IA compatible (OpenAI, Anthropic u otro proveedor soportado).

## Instalación

Inicializa AIOS en un nuevo proyecto con un solo comando:

```bash
npx @synkra/aios-core init my-project
```

Para instalar en un proyecto existente, navega al directorio del proyecto y ejecuta:

```bash
cd my-project
npx @synkra/aios-core init .
```

El instalador crea la estructura completa del framework y configura todos los archivos necesarios automáticamente.

## Estructura del Proyecto

Después de la instalación, tu proyecto tendrá la siguiente estructura:

```
my-project/
├── .aios/
│   ├── config.json          # Configuración principal
│   ├── data/                # Datos persistentes y memoria
│   └── cache/               # Caché de análisis
├── .aios-core/
│   ├── agents/              # Definiciones de los 12 agentes
│   ├── workflows/           # Workflows automatizados
│   ├── tasks/               # Definiciones de tareas
│   ├── templates/           # Plantillas de stories y documentos
│   ├── scripts/             # Scripts de automatización
│   └── docs/                # Documentación interna del framework
├── docs/
│   ├── stories/             # Stories y épicas del proyecto
│   └── qa/                  # Reportes de QA
└── .env                     # Claves de API (no commitear)
```

Los directorios `.aios/` y `.aios-core/` contienen toda la lógica del framework. El directorio `docs/` es donde se generan los artefactos del proyecto.

## Configuración

### Archivo de Configuración Principal

El archivo `.aios/config.json` controla el comportamiento del framework:

```json
{
  "project": {
    "name": "my-project",
    "type": "web-app"
  },
  "ai": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-20250514"
  },
  "features": {
    "memory": true,
    "learning": true,
    "codeReview": true
  }
}
```

### Variables de Entorno

Configura tus claves de API en el archivo `.env`:

```bash
# Proveedor de IA (obligatorio)
ANTHROPIC_API_KEY=sk-ant-...
# o
OPENAI_API_KEY=sk-...

# Opcional
GITHUB_TOKEN=ghp_...
```

**Importante:** Nunca hagas commit del archivo `.env`. Ya está en `.gitignore` por defecto.

## Entendiendo el Meta-Agente

AIOS opera como un **meta-agente** -- un sistema que coordina múltiples agentes especializados. Sus capacidades incluyen:

- **Auto-análisis:** Examina el codebase para entender patrones, dependencias y arquitectura
- **Generación de código:** Produce código siguiendo los patrones identificados en el proyecto
- **Aprendizaje de patrones:** Detecta y registra patrones recurrentes para reutilización
- **Auto-evolución:** Adapta sus estrategias basándose en el historial de decisiones

El meta-agente no es solo un asistente de código -- mantiene contexto a largo plazo, gestiona el ciclo de vida completo de stories y garantiza consistencia entre todos los agentes.

## Comandos Básicos

Todos los comandos de AIOS usan el prefijo `*` (asterisco):

| Comando | Descripción |
|---------|-------------|
| `*help` | Lista todos los comandos disponibles |
| `*status` | Muestra el estado actual del proyecto y agentes activos |
| `*config` | Muestra o modifica la configuración |
| `*list-agents` | Lista todos los agentes y su estado |
| `*activate` | Activa el pipeline de activación de un agente |
| `*analyze` | Ejecuta análisis del codebase |
| `*suggest` | Genera sugerencias de mejora |
| `*learn` | Inicia un ciclo de aprendizaje de patrones |

Para ayuda detallada sobre cualquier comando, usa `*help {comando}`.

## Sistema de Agentes

AIOS proporciona 12 agentes especializados, cada uno con responsabilidades y autoridades específicas. Para activar un agente, usa la sintaxis `@nombre-del-agente`.

### Agentes de Gestión

| Agente | Rol | Responsabilidades |
|--------|-----|-------------------|
| `@pm` | Gerente de Proyecto | Creación de épicas, pipeline de especificación, orquestación general |
| `@po` | Product Owner | Validación de stories, priorización de backlog, seguimiento de contexto |
| `@sm` | Scrum Master | Creación de stories, selección de plantillas, facilitación |

### Agentes Técnicos

| Agente | Rol | Responsabilidades |
|--------|-----|-------------------|
| `@dev` | Desarrollador | Implementación de código, commits, ejecución de tareas |
| `@architect` | Arquitecto de Sistema | Decisiones de arquitectura, selección de tecnología, evaluación de complejidad |
| `@data-engineer` | Ingeniero de Datos | Diseño de esquemas, optimización de queries, migraciones |
| `@devops` | Ingeniero DevOps | Git push, PRs, CI/CD, gestión de MCP |

### Agentes de Calidad y Análisis

| Agente | Rol | Responsabilidades |
|--------|-----|-------------------|
| `@qa` | Aseguramiento de Calidad | Gate de calidad, revisión de código, pruebas |
| `@analyst` | Analista de Negocios | Investigación, análisis de requisitos, reportes |
| `@ux-design-expert` | Diseñador UX | Especificaciones de frontend, evaluación de interfaces |

### Agente de Gobernanza

| Agente | Rol | Responsabilidades |
|--------|-----|-------------------|
| `@aios-master` | Gobernador del Framework | Gobernanza constitucional, puede ejecutar cualquier tarea, mediación de conflictos |

## Capa de Memoria

AIOS posee un sistema de memoria persistente que soporta búsqueda semántica y por palabras clave. La memoria almacena decisiones, patrones aprendidos y contexto del proyecto.

| Comando | Descripción |
|---------|-------------|
| `*memory status` | Verifica el estado de la memoria y estadísticas |
| `*memory search {query}` | Busca en la memoria términos relevantes |
| `*memory rebuild` | Reconstruye los índices de memoria |
| `*memory clear-cache` | Limpia el caché sin eliminar datos persistentes |

La memoria soporta dos modos de búsqueda:

- **Búsqueda semántica:** Encuentra contenido por significado, incluso con términos diferentes
- **Búsqueda por palabras clave:** Coincidencia exacta para términos específicos

## Buenas Prácticas

1. **Empieza en pequeño:** Comienza con un agente (`@dev`) y expande gradualmente hacia workflows completos
2. **Usa nombres descriptivos:** Stories, épicas y branches con nombres claros facilitan la navegación
3. **Aprovecha la memoria:** Ejecuta `*analyze` regularmente para mantener actualizado el contexto del proyecto
4. **Analiza frecuentemente:** Usa `*suggest` para identificar oportunidades de mejora continua
5. **Sigue los workflows:** El ciclo completo (crear, validar, implementar, revisar) existe para garantizar calidad
6. **Documenta decisiones:** El log de decisiones (`decision-log`) es fundamental para la trazabilidad

## Solución de Problemas

### Node.js no encontrado o versión incorrecta

```bash
# Verifica la versión instalada
node --version

# Si es menor que 18, actualiza vía nvm
nvm install 18
nvm use 18
```

### Error de clave de API

Si recibes errores de autenticación, verifica:

1. El archivo `.env` existe en la raíz del proyecto
2. La clave está en el formato correcto para el proveedor elegido
3. La clave tiene créditos/cuota disponible

### Problemas de memoria

Si la memoria está lenta o retorna resultados incorretos:

```bash
# Reconstruye los índices
*memory rebuild

# Si persiste, limpia el caché
*memory clear-cache
```

### Error en la inicialización

Si `npx @synkra/aios-core init` falla:

```bash
# Limpia el caché de npm
npm cache clean --force

# Intenta de nuevo con la flag verbose
npx @synkra/aios-core init my-project --verbose
```
