# Desarrollo de Historias

Una guía completa del proceso de desarrollo impulsado por historias en AIOS.

## Ciclo de Vida de las Historias

Las historias siguen una progresión de estado estricta:

```
Borrador → Listo → EnProgreso → EnRevisión → Completado
```

| Estado | Agente | Disparador |
|--------|--------|-----------|
| Borrador | @sm | Historia creada |
| Listo | @po | Validación pasada (OK) |
| EnProgreso | @dev | Implementación iniciada |
| EnRevisión | @qa | Revisión de puerta QA |
| Completado | @devops | Push a remoto |

## Fase 1: Crear (@sm)

El Maestro Scrum redacta historias a partir de épicas usando el comando `*draft`.

**Salida:** `{epicNum}.{storyNum}.story.md`

## Fase 2: Validar (@po)

El Propietario del Producto valida usando una lista de verificación de 10 puntos:

1. Título claro y objetivo
2. Descripción completa
3. Criterios de aceptación comprobables
4. Alcance bien definido (DENTRO/FUERA)
5. Dependencias mapeadas
6. Estimación de complejidad
7. Valor comercial definido
8. Riesgos documentados
9. Criterios de Hecho
10. Alineación con PRD/Epic

**Decisión:** OK (≥7/10) o NO OK

## Fase 3: Implementar (@dev)

Tres modos de ejecución:

- **YOLO** — Autónomo, mínimas indicaciones
- **Interactivo** — Por defecto, puntos de control educativos
- **Pre-Vuelo** — Todas las preguntas antes, luego ejecutar

## Fase 4: Puerta QA (@qa)

Siete controles de calidad:

1. Revisión de código
2. Pruebas unitarias
3. Criterios de aceptación cumplidos
4. Sin regresiones
5. Rendimiento aceptable
6. Seguridad verificada
7. Documentación actualizada

**Veredictos:** APROBADO | OBSERVACIONES | RECHAZO | DISPENSADO

## Estructura de Archivo de Historia

```
docs/stories/
├── {epicNum}.{storyNum}.story.md
└── {epicNum}.{storyNum}/
    ├── spec/
    │   └── spec.md
    └── plan/
        └── implementation.yaml
```
