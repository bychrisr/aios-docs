# Ejecución de Flujos de Trabajo

Cómo los flujos de trabajo de AIOS orquestan el desarrollo multi-agente.

## Principio Central

Los flujos de trabajo están compuestos de **tareas conectadas**, no agentes. Cada tarea define entradas, salidas, pre/post-condiciones y modos de ejecución. Los agentes son los ejecutores por defecto.

## Flujos de Trabajo Disponibles

| Flujo de Trabajo | Propósito |
|------------------|-----------|
| Ciclo de Desarrollo de Historias | Flujo de trabajo de desarrollo principal |
| Bucle QA | Ciclo iterativo de revisión-corrección |
| Canalización de Especificaciones | Requisitos → especificación ejecutable |
| Descubrimiento Brownfield | Evaluación de codebases heredadas |

## Selección de Flujo de Trabajo

| Situación | Flujo de Trabajo |
|-----------|-----------------|
| Nueva historia de epic | Ciclo de Desarrollo de Historias |
| QA encontró problemas | Bucle QA |
| Característica compleja necesita spec | Canalización de Especificaciones → SDC |
| Unirse a proyecto existente | Descubrimiento Brownfield |
| Corrección de error simple | SDC (modo YOLO) |

## Modos de Ejecución

### Modo Guiado
Cambio manual de persona entre agentes. El orquestador te dice qué agente activar a continuación.

### Modo Motor
Generación de subagenetes real — los agentes se distribuyen automáticamente en secuencia.

## Iniciando un Flujo de Trabajo

```
@aios-master
> *workflow greenfield-fullstack
```

O para flujos de trabajo específicos:

```
> *workflow story-development-cycle
> *workflow brownfield-discovery
```

## Configuración del Flujo de Trabajo

Los flujos de trabajo se definen como archivos YAML en `.aios-core/development/workflows/`. Cada flujo de trabajo especifica:

- **Fases** con pasos secuenciales
- **Asignaciones de agentes** por paso
- **Artefactos** producidos en cada fase
- **Condiciones de puerta** para progresión
