# Story Development

A complete guide to the story-driven development process in AIOS.

## Story Lifecycle

Stories follow a strict status progression:

```
Draft → Ready → InProgress → InReview → Done
```

| Status | Agent | Trigger |
|--------|-------|---------|
| Draft | @sm | Story created |
| Ready | @po | Validation passed (GO) |
| InProgress | @dev | Implementation started |
| InReview | @qa | QA gate review |
| Done | @devops | Push to remote |

## Phase 1: Create (@sm)

The Scrum Master drafts stories from epics using the `*draft` command.

**Output:** `{epicNum}.{storyNum}.story.md`

## Phase 2: Validate (@po)

The Product Owner validates using a 10-point checklist:

1. Clear and objective title
2. Complete description
3. Testable acceptance criteria
4. Well-defined scope (IN/OUT)
5. Dependencies mapped
6. Complexity estimate
7. Business value defined
8. Risks documented
9. Criteria of Done
10. Alignment with PRD/Epic

**Decision:** GO (≥7/10) or NO-GO

## Phase 3: Implement (@dev)

Three execution modes:

- **YOLO** — Autonomous, minimal prompts
- **Interactive** — Default, educational checkpoints
- **Pre-Flight** — All questions upfront, then execute

## Phase 4: QA Gate (@qa)

Seven quality checks:

1. Code review
2. Unit tests
3. Acceptance criteria met
4. No regressions
5. Performance acceptable
6. Security verified
7. Documentation updated

**Verdicts:** PASS | CONCERNS | FAIL | WAIVED

## Story File Structure

```
docs/stories/
├── {epicNum}.{storyNum}.story.md
└── {epicNum}.{storyNum}/
    ├── spec/
    │   └── spec.md
    └── plan/
        └── implementation.yaml
```
