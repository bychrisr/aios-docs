# Learning Trails by Role

Personalized learning paths based on your role in the team. Each trail is a curated sequence of topics that takes you from basics to proficiency in the areas most relevant to your work.

---

## New User Trail

**For:** Anyone new to Synkra AIOS | **Time:** 1-2 hours

Start here if you have never used AIOS before. This trail covers installation, core concepts, and your first workflow.

1. [Quick Start Guide](/playbook/getting-started/quick-start) -- Install AIOS and run your first command
2. [Onboarding in 60 Minutes](/playbook/getting-started/onboarding-60min) -- Structured walkthrough of all key features
3. [Introduction to AIOS](/docs/guides/getting-started) -- Understand the meta-agent and framework concepts
4. [The 12 AIOS Agents](/docs/agents) -- Meet the specialized agents and their roles
5. [Command Reference](/playbook/commands) -- Browse available commands

**After completing this trail,** choose a role-specific trail below to deepen your knowledge.

---

## Developer Trail

**For:** Software developers and engineers | **Agent focus:** @dev (Dex)

Learn how to use AIOS for daily development work: activating agents, implementing stories, running quality checks, and shipping code.

1. [Agent Activation](/docs/guides/agent-activation) -- How to activate and switch between agents
2. [Story Development Cycle](/docs/workflows/story-development-cycle) -- The primary 4-phase development workflow
3. [Command Reference](/playbook/commands) -- Commands you will use daily (`*develop`, `*status`, `*diff`)
4. [QA Loop](/docs/workflows/qa-loop) -- Iterative review-fix cycle after implementation
5. [PR Workflow](/playbook/workflows/pr-workflow) -- From local changes to merged pull request
6. [Validation Checklists](/playbook/checklists) -- Quality standards your code must meet
7. [Template Reference](/playbook/templates) -- Templates for stories and documentation

**Key commands to learn:**
- `@dev *develop {story-id}` -- Start story implementation
- `@qa *qa-gate` -- Run quality checks
- `@devops *push` -- Push changes to remote (delegated)

**Execution modes:** Interactive (default), YOLO (autonomous), Pre-Flight (plan-first)

## QA Engineer Trail

**For:** Quality assurance engineers and testers | **Agent focus:** @qa

Learn how to use AIOS quality gates, the QA loop, and CodeRabbit integration to ensure code quality.

1. [Story Development Cycle](/docs/workflows/story-development-cycle) -- Understand the full cycle (focus on Phase 4: QA Gate)
2. [QA Loop](/docs/workflows/qa-loop) -- Automated review-fix cycle (up to 5 iterations)
3. [Validation Checklists](/playbook/checklists) -- QA Gate 7-point checklist and verdict criteria
4. [System Architecture](/docs/architecture) -- Understand the system you are testing
5. [Command Reference](/playbook/commands) -- QA-specific commands

**Key commands to learn:**
- `@qa *qa-gate` -- Run the 7-point quality gate
- `*qa-loop {storyId}` -- Start automated review-fix loop
- `*stop-qa-loop` -- Pause the loop
- `*resume-qa-loop` -- Resume from saved state
- `*escalate-qa-loop` -- Force manual escalation

**QA gate verdicts:** PASS, CONCERNS, FAIL, WAIVED

**Escalation triggers:** `max_iterations_reached`, `verdict_blocked`, `fix_failure`, `manual_escalate`
