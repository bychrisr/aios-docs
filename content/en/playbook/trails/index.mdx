# Trails by Role

Curated learning paths based on your role. Each trail links to verified content — no duplicated documentation.

---

## New User Trail

**For:** Anyone new to Synkra AIOS | **Time:** 1-2 hours

Start here if you have never used AIOS before.

1. [Quick Start Guide](/playbook/getting-started/quick-start) — Install AIOS and run your first command (5 steps, under 10 min)
2. [Onboarding in 60 Minutes](/playbook/getting-started/onboarding-60min) — Complete walkthrough: all 12 agents, SDC, quality gates
3. [Agents at a Glance](/playbook/agents) — Meet all agents, their names, and commands
4. [Cheat Sheet](/playbook/cheat-sheet) — Copy-paste sequences for daily use

After completing this trail, choose a role-specific trail below.

---

## Developer Trail

**For:** Software developers | **Agent focus:** @dev (Dex)

Learn daily AIOS development: activating agents, implementing stories, running quality checks, shipping code.

1. [Agents at a Glance](/playbook/agents) — Focus on Dex (@dev), Gage (@devops), Quinn (@qa)
2. [Story Development Cycle](/docs/guides/workflows/STORY-DEVELOPMENT-CYCLE-WORKFLOW) — The primary 4-phase workflow
3. [Specialized Commands](/playbook/commands/specialized-commands) — @dev, @qa, and @devops commands
4. [QA Loop](/docs/guides/workflows/QA-LOOP-WORKFLOW) — Automated review-fix cycle
5. [Validation Checklists](/playbook/checklists) — Quality standards your code must meet
6. [Cheat Sheet](/playbook/cheat-sheet) — Daily command sequences

**Key commands to learn:**
- `@dev *develop {story-id}` — Start implementation (Interactive, YOLO, or Pre-Flight mode)
- `@qa *qa-gate` — Run the 7-point quality gate
- `@devops *push` — Ship (delegated, only Gage can push)

## Project Manager Trail

**For:** Project managers and product managers | **Agent focus:** @pm (Morgan)

Learn to manage epics, run the spec pipeline, and coordinate sprints.

1. [Agents at a Glance](/playbook/agents) — Focus on Morgan (@pm), River (@sm), Pax (@po)
2. [Spec Pipeline](/docs/guides/workflows/SPEC-PIPELINE-WORKFLOW) — Transform requirements into executable specs
3. [Specialized Commands](/playbook/commands/specialized-commands) — @pm, @sm commands
4. [Validation Checklists](/playbook/checklists) — PM/Change Management checklist and complexity scoring
5. [Template Reference](/playbook/templates) — PRD and documentation templates

**Key commands to learn:**
- `@pm *create-epic` — Create a new epic
- `@pm *execute-epic` — Execute an epic plan
- `@pm *create-doc prd` — Generate a PRD

**Spec Pipeline complexity classes:** SIMPLE (≤ 8 pts), STANDARD (9-15), COMPLEX (≥ 16)

## QA Engineer Trail

**For:** Quality assurance engineers | **Agent focus:** @qa (Quinn)

Learn quality gates, the QA loop, and CodeRabbit integration.

1. [Agents at a Glance](/playbook/agents) — Focus on Quinn (@qa)
2. [Story Development Cycle](/docs/guides/workflows/STORY-DEVELOPMENT-CYCLE-WORKFLOW) — Focus on Phase 4: QA Gate
3. [QA Loop](/docs/guides/workflows/QA-LOOP-WORKFLOW) — Automated review-fix cycle (up to 5 iterations)
4. [Validation Checklists](/playbook/checklists) — 7-point QA gate checklist, verdict criteria, CodeRabbit severity
5. [Specialized Commands](/playbook/commands/specialized-commands) — @qa commands

**Key commands to learn:**
- `@qa *qa-gate` — 7-point quality gate
- `*qa-loop {storyId}` — Automated review-fix loop
- `*stop-qa-loop` / `*resume-qa-loop` — Pause and resume
- `*escalate-qa-loop` — Force manual escalation

**Gate verdicts:** PASS, CONCERNS, FAIL, WAIVED

## Architect Trail

**For:** Tech leads and software architects | **Agent focus:** @architect (Aria)

Learn system design, brownfield assessment, and technical guidance using AIOS.

1. [Agents at a Glance](/playbook/agents) — Focus on Aria (@architect), Dara (@data-engineer), Atlas (@analyst)
2. [Spec Pipeline](/docs/guides/workflows/SPEC-PIPELINE-WORKFLOW) — Complexity assessment and implementation planning
3. [Validation Checklists](/playbook/checklists) — Architecture review checklist
4. [Master Agent Commands](/playbook/commands/master-agent-commands) — IDS commands for REUSE/ADAPT/CREATE decisions

**Key commands to learn:**
- `@architect *create-full-stack-architecture` — Generate full architecture docs
- `@architect *execute-checklist architect-checklist` — Architecture review
- `*ids check {intent}` — Should this be REUSE, ADAPT, or CREATE?

**Brownfield Discovery phases:** Data Collection (1-3), Draft and Validation (4-7), Finalization (8-10)
