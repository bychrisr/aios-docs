# Checklists

Checklists de qualidade para cada fase do desenvolvimento com AIOS.

## Validação de Story (10 Pontos)

1. Título claro e objetivo
2. Descrição completa (problema/necessidade explicados)
3. Critérios de aceitação testáveis (Given/When/Then preferível)
4. Escopo bem definido (IN e OUT claramente listados)
5. Dependências mapeadas (stories/recursos pré-requisito)
6. Estimativa de complexidade (pontos ou T-shirt sizing)
7. Valor de negócio (benefício para o usuário/negócio claro)
8. Riscos documentados (problemas potenciais identificados)
9. Critérios de Done (definição clara de conclusão)
10. Alinhamento com PRD/Epic (consistência com documentos fonte)

**Nota de Aprovação:** ≥ 7/10

## QA Gate (7 Verificações)

1. Code review — padrões, legibilidade, manutenibilidade
2. Testes unitários — cobertura adequada, todos passando
3. Critérios de aceitação — todos atendidos conforme AC da story
4. Sem regressões — funcionalidades existentes preservadas
5. Performance — dentro dos limites aceitáveis
6. Segurança — básicos do OWASP verificados
7. Documentação — atualizada se necessário
