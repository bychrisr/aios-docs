# Comandos Especializados por Agente

Cada agente AIOS tem seu próprio conjunto de comandos para seu domínio. Use o handle do agente para endereçá-los diretamente.

Para uma referência rápida em uma página única, veja o [Cheat Sheet](/playbook/cheat-sheet). Para personas e papéis dos agentes, veja [Agentes](/playbook/agents).

---

## @sm — River (Scrum Master)

| Comando | Descrição |
|---------|-----------|
| `@sm *draft` | Criar uma nova story (criação guiada interativa) |
| `@sm *create-story` | Comando alternativo de criação de story |
| `@sm *help` | Mostrar todos os comandos de @sm |

---

## @po — Pax (Product Owner)

| Comando | Descrição |
|---------|-----------|
| `@po *validate-story-draft` | Executar checklist de validação de 10 pontos; decisão GO/NO-GO |
| `@po *help` | Mostrar todos os comandos de @po |

**Decisão:** GO (≥ 7/10) move a story de `Draft` para `Ready`.

---

## @dev — Dex (Desenvolvedor)

| Comando | Descrição |
|---------|-----------|
| `@dev *develop {story-id}` | Iniciar implementação de uma story (modo Interactive) |
| `@dev *develop {story-id} --yolo` | Modo YOLO — autônomo, 0-1 prompts |
| `@dev *develop {story-id} --pre-flight` | Pre-Flight — todas as perguntas antecipadas |
| `@dev *help` | Mostrar todos os comandos de @dev |

**Autoridade:** Pode `git add`, `git commit`, `git branch`, `git checkout`. Não pode `git push` — delegar ao @devops.

---

## @qa — Quinn (Engenheiro de QA)

| Comando | Descrição |
|---------|-----------|
| `@qa *qa-gate` | Executar quality gate de 7 pontos |
| `*qa-loop {storyId}` | Iniciar ciclo automatizado de revisão-correção (máx. 5 iterações) |
| `*qa-loop-review` | Retomar o ciclo a partir da etapa de revisão |
| `*qa-loop-fix` | Retomar o ciclo a partir da etapa de correção |
| `*stop-qa-loop` | Pausar o ciclo e salvar estado em `qa/loop-status.json` |
| `*resume-qa-loop` | Retomar do estado salvo |
| `*escalate-qa-loop` | Forçar escalação manual |
| `@qa *help` | Mostrar todos os comandos de @qa |

**Veredictos do QA gate:** PASS, CONCERNS, FAIL, WAIVED

## Regras de Delegação

O AIOS aplica padrões estritos de delegação. Regras principais:

- Qualquer agente que precise fazer push de código DEVE delegar para `@devops *push`
- @architect decide a tecnologia; @data-engineer implementa o DDL
- @sm cria stories; @po as valida — nenhum pode fazer o trabalho do outro
- @aios-master (Orion) pode sobrepor limites quando necessário para a saúde do framework
