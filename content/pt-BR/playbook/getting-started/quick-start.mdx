# Guia de Início Rápido

Comece a usar o AIOS em 5 passos.

## Passo 1: Inicializar o AIOS

Na raiz do seu projeto:

```bash
npx @synkra/aios-core init
```

Isso configura o diretório `.aios-core/` com todos os arquivos do framework, agentes, tasks e workflows.

## Passo 2: Ativar Seu Primeiro Agente

No Claude Code (ou sua IDE de IA):

```
@dev
```

Isso ativa Dex, o agente Desenvolvedor, com todas as capacidades de desenvolvimento. Você pode endereçar qualquer um dos 12 agentes pelo seu handle.

## Passo 3: Executar Seu Primeiro Comando

```
*help
```

Mostra todos os comandos disponíveis para o agente ativo. Todo agente suporta `*help`, `*status` e `*guide`.

## Passo 4: Criar Sua Primeira Story

Ative River, o Scrum Master, e crie uma story:

```
@sm *draft
```

Siga os prompts interativos para criar sua primeira story de desenvolvimento. A saída é um arquivo `{epicNum}.{storyNum}.story.md` com status `Draft`.

Em seguida, valide-a com Pax, o Product Owner:

```
@po *validate-story-draft
```

Uma nota ≥ 7/10 move a story de `Draft` para `Ready`.

## Passo 5: Começar a Desenvolver

Com uma story no status `Ready`:

```
@dev *develop story-1.1
```

Dex guia você pela implementação no modo Interactive por padrão. O CodeRabbit escaneia e corrige automaticamente problemas CRITICAL e HIGH.

Quando a implementação estiver completa:

```
@qa *qa-gate
```

Quinn executa o quality gate de 7 pontos. No PASS, entregue com:

```
@devops *push
```

Apenas Gage (@devops) pode fazer push — todos os outros agentes delegam operações de push para ele.

---

## Comandos Essenciais de Referência Rápida

| Comando | Agente | Descrição |
|---------|--------|-----------|
| `*help` | Qualquer | Mostrar comandos disponíveis |
| `@sm *draft` | River | Criar uma nova story |
| `@po *validate-story-draft` | Pax | Validar uma story (checklist de 10 pontos) |
| `@dev *develop {id}` | Dex | Iniciar implementação de uma story |
| `@qa *qa-gate` | Quinn | Executar verificações de qualidade (gate de 7 pontos) |
| `@devops *push` | Gage | Fazer push das alterações para o remoto |
